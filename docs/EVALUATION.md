
# 評価計画（定量評価・定性評価）

本ドキュメントは、ChordLens Web の評価実験（定量評価・定性評価）の手順、取得データ、評価指標、アンケート設問案をまとめたものである。

## 0. 前提（実装と取得可能なデータ）

- 本システムは Web Audio API によるリアルタイム解析を行い、**周波数と純正律期待値から cent（セント）偏差を算出**して評価する。
- 実験用ページ（`/experiment`）では、解析結果を時系列ログとして記録・CSV出力できる。
- **音声波形そのものの録音・保存は行わない**（ログは解析結果ベース）。

### 取得できる主データ（例）

- `centDeviation`：期待周波数に対する実測周波数のセント偏差
- 各構成音（音名）ごとの時系列データ（一定間隔）
- 実験条件（A/B）、課題（和音種別、テンポ、拍数など）をログ/メタデータとして紐付け

## 1. 定量評価（A/B実験）

### 1.1 目的

システム（視覚フィードバック）を用いることで、和音練習の**効率化（短時間での改善）**に効果があるかを検証する。

### 1.2 仮説

- H1：システムあり条件は、システムなし条件よりも、練習後の $|centDeviation|$ が小さい。
- H2：システムあり条件は、システムなし条件よりも、目標範囲（例：±10 cent）への到達が速い。
- H3：システムあり条件は、システムなし条件よりも、cent偏差の時間変動（分散/IQR）が小さい。

### 1.3 実験デザイン（推奨）

#### 被験者間A/B（ペア単位の割付）

本研究では、被験者を2人1組のペア（同一楽器、または音域の近い楽器）にし、**ペアごとに A/B を割り付ける**ことを想定する。

- 例：
	- ペア1（Tp, Tp）：A（システムあり）
	- ペア2（Tb, Hr）：B（システムなし）

このデザインは、同時演奏や環境条件をペア内で揃えやすい一方、統計的には「ペアにクラスタ（相関）がある被験者間比較」になるため、解析時に考慮する（1.9参照）。

##### ペア構成のルール

1. **優先：同一楽器ペア**（Tp–Tp、Tb–Tb など）を基本とする。
2. **補完：異楽器ペア**は、同一楽器でペアが組めない場合に限り、音域の近い楽器（例：Tb–Hr、Ob–Cl）で構成する。
3. **ペアタイプの記録**：各ペアが「同一楽器 / 異楽器（金管同士）/ 異楽器（木管同士）」等を記録し、解析時の層別または共変量として扱う。

#### 参考：クロスオーバー（被験者内比較）

もし実施可能であれば、同一被験者が A/B 両条件を体験するクロスオーバーは個人差を強く相殺できる。ただし本研究の主計画はペア単位割付とする。

### 1.4 実験条件の統制（重要）

- **根音提示の扱い**：
	- 根音ドローン（基準音提示）を使う場合、A/Bで同一条件にする（例：外部メトロノーム/ドローンアプリを両条件で使用）。
	- A条件のみ根音提示を行うと、視覚フィードバックの効果と混ざるため避ける。
- **ペアタイプの層別化（推奨）**：
	- ペア構成（同一楽器/異楽器）が条件間で偏ると、システムの効果と楽器特性の交絡が生じる。
	- 可能な限り、同一楽器ペアを A/B 両群に均等に配置し、異楽器ペアも同様に配置する（層別ランダム化）。
	- 例：同一楽器ペア4組 → 2組ずつA/B、異楽器ペア2組 → 1組ずつA/B
- 解析設定：A4基準、閾値、FFT等を実験中は固定する。
- 演奏課題：まずは**3音（triad）中心**とし、多声音（5音/7音）は探索的評価に留める（ピーク混入で解析が不安定になり得るため）。

### 1.5 被験者

- 対象：管弦楽器・声楽など（和音練習を行う可能性がある奏者）
- 収集する背景要因：年齢、演奏歴、練習頻度、純正律の理解度、音程評価の自己効力感など

### 1.6 課題（テスト課題）

- 被験者に演奏する音を指定し、テンポ60で4拍（または8拍）維持させる。
- 和音は、根音＋指定構成音（例：長三和音）で実施する。
- 各試行は、同じ構成で「練習前テスト」「練習後テスト」を行う。

### 1.7 手順（例：1セッション）

1. 実験前アンケート（背景・共変量）
2. 練習前テスト（Test 1）
    - 課題の説明・デモ（短時間）
    - 指定和音を演奏（テンポ60、4拍）
    - 解析ログを記録（CSV出力用）
3. 10分間練習
    - A条件：視覚フィードバックあり
    - B条件：視覚フィードバックなし（ログ取得のみ行う場合は表示を無効化）
4. 練習後テスト（Test 2）
    - Test 1と同一課題を実施
    - 解析ログを記録
5. 実験後アンケート（主観評価）

※クロスオーバーの場合は、休憩（例：5分）を挟んで条件を入れ替えて 2〜5 を繰り返す。

### 1.8 評価指標（ログから算出）

ログの `centDeviation` を用い、以下を主指標とする。

- **精度（Accuracy）**：$|centDeviation|$ の平均/中央値（各構成音・各試行）
- **安定性（Stability）**：$centDeviation$ の分散、IQR（四分位範囲）
- **到達時間（Time-to-Target）**：$|centDeviation| \le 10$ cent を連続で X ms 満たすまでの時間
- **成功率（In-Target Rate）**：評価区間における閾値内フレーム割合

補足：アタック直後は不安定になりやすいため、「後半2拍のみ評価」等、評価区間のルールを事前固定する。

### 1.9 統計・解析（簡易方針）

- **主解析**：各被験者の改善量（Test2 − Test1、または改善率）を算出し、A/Bで比較する。
- **ペア単位割付の考慮**：同一ペア内の2名は独立ではない可能性があるため、可能なら以下のいずれかで扱う。
	- ペアをクラスタとして扱い、**ペア平均（またはペア中央値）でA/B比較**する（簡易）。
		- 例：ペアiの改善量 = (被験者i1の改善量 + 被験者i2の改善量) / 2
		- このペア平均をA群・B群で集計し、Mann-Whitney U検定やWelchのt検定で比較。
	- 線形混合モデル等で「ペア」をランダム効果として扱う（厳密）。
- **ペアタイプ（同一楽器/異楽器）の扱い**：
	- 層別化が不完全な場合、ペアタイプを共変量として記述統計に反映するか、層別解析を補助的に実施する。
	- 報告例：「同一楽器ペア：A群平均=X, B群平均=Y / 異楽器ペア：A群平均=Z, B群平均=W」
- **報告内容**：平均±SD、中央値[IQR]、効果量（Cohen's d、Cliff's delta等）、ペア数（クラスタ数）、個人数を併記する。

## 2. 定量評価用アンケート（設問案）

### 2.1 実験前アンケート（背景）

尺度：特記なければ 1〜7（1=全くそう思わない、7=非常にそう思う）

1. 年齢（任意）：＿＿歳
2. 主な担当楽器（声楽の場合は声種）：＿＿
3. 演奏歴：＿＿年＿＿ヶ月
4. 直近1ヶ月の練習頻度：週＿＿日、1日平均＿＿分
5. 音程（ピッチ）を耳で合わせるのは得意だと思う（1〜7）
6. 和音を聴いて「合っている/濁っている」が分かる（1〜7）
7. 純正律という概念を知っている（0:知らない / 1:聞いたことがある / 2:理解している / 3:演奏で意識している）
8. 今日の体調は良い（1〜7）
9. 今日の集中状態は良い（1〜7）

### 2.2 実験後アンケート（主観評価）

尺度：1〜7（1=全くそう思わない、7=非常にそう思う）

#### 効果（効率）

1. 10分の練習で、音程（和音の合い方）が改善したと感じる
2. どこを直せば良いか（どの音が高い/低いか）が明確だった
3. 修正→再試行のサイクルを短く回せた
4. 練習の「迷い」が減った

#### 負担・集中

5. 練習中の精神的負担が大きかった
6. 注意が散って演奏に集中しづらかった
7. 疲労感が強かった

#### 信頼・理解

8. 表示/フィードバックは自分の感覚と一致していた
9. 結果を信用できると感じた
10. フィードバックの意味（ズレの方向・量）を理解できた

#### 自由記述

11. 良かった点（自由記述）：＿＿
12. 困った点・改善案（自由記述）：＿＿

## 3. 定性評価（使用感評価）

### 3.1 目的

パート練習・和音練習の実運用の中で、システムの**有用性・使いやすさ・理解容易性・継続利用意向**を定性的に把握する。

### 3.2 実施方法（例）

- 被験者に一定期間（例：1回20分×複数回）使ってもらい、使用後にアンケートに回答してもらう。
- 使用場面は自由だが、可能なら以下を含める：
	- 個人練習での和音合わせ
	- パート内の和音合わせ
	- 合奏前のピッチ確認

## 4. 定性評価用アンケート（設問案）

尺度：1〜7（1=全くそう思わない、7=非常にそう思う）

### 4.1 有用性（Usefulness）

1. 和音練習の質が上がると感じた
2. 音程のズレに気づく頻度が増えた
3. 自分だけでは分からない問題を発見できた
4. 指導（先生/パートリーダー）にも有用だと思う

### 4.2 使いやすさ（Usability）

5. 操作は直感的で迷わなかった
6. 初めてでも短時間で使い方を理解できた
7. 画面の情報量は適切だった
8. 表示更新が速く、ストレスが少ない

### 4.3 フィードバック理解・学習支援

9. 「どの音が」「どれだけ」ズレているかが分かりやすい
10. ズレを直す行動に結びついた
11. 使うほど耳が鍛えられると感じた

### 4.4 運用適合性

12. 個人練習で使いたい
13. パート練習で使いたい
14. 合奏前の和音合わせで使いたい
15. 準備（マイク許可・設定等）が面倒だと感じた

### 4.5 自由記述（具体例を促す）

16. 良かった点（具体的な場面とともに）：＿＿
17. 困った点・誤解した点（具体的な場面とともに）：＿＿
18. 改善してほしい点（優先度が高い順に）：＿＿
19. 使えると思う場面／使いにくい場面：＿＿

### 4.6 推薦意向（NPS）

20. 他の奏者にも勧めたい（0〜10）：＿＿

## 5. 倫理・プライバシー（運用メモ）

- 収集データは解析結果ログ（セント偏差等）とアンケート回答であり、音声波形は保存しない。
- 研究参加は任意であること、途中撤回可能であること、データの取扱い（匿名化・保存期間）を事前に説明する。
